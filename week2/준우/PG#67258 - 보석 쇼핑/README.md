# PG#67258 - 보석 쇼핑

## 느낀점
* 효율성을 어떻게 가져갈지 모르겠다.

## input
```
gems : ["DIA", "RUBY", "RUBY", "DIA", "DIA", "EMERALD", "SAPPHIRE", "DIA"]
```

## output
```
[3, 7]
```

## notes
* 다 들어가게, 범위 좁혀서, 시작점 작게
* 1부터 count

## strategy
* 구현 자체는 할 수 있을 것 같다
  * 인덱스 0부터 다 채울때
  * 인덱스 1부터 다 채울때
  * ...
  * 인덱스 n-1부터 다 채울때
  * 중에서 최솟값.
* 그래서 정확성은 맞췄는데, 어떻게 효율성까지 해결할까?
  * 다른 자료구조?
  * 정렬 후 이진탐색?
  * 흠..
