# PG#42576 - 완주하지 못한 선수

## 느낀점
* 예전에 이 문제를 태현이와 풀었을때는 이게 왜 해시테이블 문제인지, 왜 시간초과가 뜨는지 몰랐었다.
* 이젠 알겠다.
  * 파이썬 리스트(동적 배열)에서의 in은 O(n)이 걸리는 반면 
  * 파이썬 딕셔너리(해시테이블)에서의 in은 O(1)이 걸림.
* 카운터는 이럴 때 유용하게 쓰인다.

```
participants : ["leo", "kiki", "eden"] // 세명이 달렸는데

```

## output
```
"leo" // 골인하지 못한 한명을 찾아야 하는 문제
```
